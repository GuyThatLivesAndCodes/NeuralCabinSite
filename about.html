About NeuralCabin

A workbench, not a chat box.

NeuralCabin is a pure Rust neural network workbench. You build networks from scratch — feed-forward for image classification, llama-style language models that export GGUF for LM Studio & Ollama. Thousands of lines of code. Real results. No bullshit.

▌ Positioning

Not another LM runner.

Most "AI tools" download somebody else's weights and point you at a chat box. NeuralCabin is the opposite: a workbench where you wire the architecture, train it on your corpus, and walk away with a model that's yours.

▢ LM runners

Run someone else's model.

  • —Pick a model from a list. Download gigabytes of weights.
  • —Type into a chat box. Hope it does the thing.
  • —The architecture, the data, the training are opaque.
  • —You are a consumer of the model.
▌ NeuralCabin

You're the author.

  • +Compose layers, activations, losses, optimizers by hand.
  • +Bring your own corpus. Train. Watch the loss in real time.
  • +Export GGUF for LM Studio & Ollama, or JSON checkpoints.
  • +You own every weight. You understand every line.
▌ Engine

Thousands of lines. Zero math deps.

The engine is hand-written Rust. Tensors, autograd, optimizers — all of it. No ndarray, no tch-rs, no candle. You can read the whole stack in an afternoon.

01engine/tensor.rs

Dense tensors & tape-based autograd

Reverse-mode automatic differentiation on a flat Vec<f32> tensor type. Gradient checks ship with the repo so you can audit every backward pass.

The classics, implemented end-to-end. XOR MLP converges to <1e-3 loss in under 2k epochs with no handholding.

▌ API

An XOR network in 20 lines.

The engine is a normal Rust crate — pull it into any project. No build daemon, no protobuf, no python. The same code the workbench runs under the hood.

Composable, auditable, fast enough.

Define a model as a flat list of layer specs. Pick an optimizer and a loss. Hand it tensors. The autograd tape records every operation; train_step walks it backwards.

There is nothing magic happening underneath. You can cargo expand it and you will recognize every line.

~3,400 LINES ENGINE RUST · 13 TESTS · 0 MATH CRATES
examples/xor.rs
use neuralcabin_engine::{
    nn::{LayerSpec, Model},
    optimizer::{Optimizer, OptimizerKind},
    tensor::Tensor,
    Activation, Loss,
};

let mut model = Model::from_specs(2, &[
    LayerSpec::Linear { in_dim: 2, out_dim: 8 },
    LayerSpec::Activation(Activation::Tanh),
    LayerSpec::Linear { in_dim: 8, out_dim: 1 },
    LayerSpec::Activation(Activation::Sigmoid),
], 42);

let mut opt = Optimizer::new(
    OptimizerKind::Adam {
        lr: 0.05, beta1: 0.9,
        beta2: 0.999, eps: 1e-8,
    },
    &model.parameter_shapes(),
);

let x = Tensor::new(vec![4,2], vec![0.,0., 0.,1., 1.,0., 1.,1.]);
let y = Tensor::new(vec![4,1], vec![0., 1., 1., 0.]);

for _ in 0..2000 {
    model.train_step(&mut opt, Loss::MeanSquaredError, &x, &y);
}

// → predictions: [0.04, 0.96, 0.96, 0.04]
let pred = model.predict(&x);
